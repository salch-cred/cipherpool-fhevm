## What we borrowed (winning patterns, adapted to a new domain)

1. **Async public-decrypt + signature-verification flow** (`FHE.makePubliclyDecryptable` -> off-chain KMS decrypt -> `FHE.checkSignatures` in a `fulfill*` callback) -- this is the exact pattern Confidential Derivatives uses for `closePosition`, `liquidatePosition`, and options exercise. CipherTrust now uses it for `requestSlashCheck` / `fulfillSlashCheck` and `requestTierReveal` / `fulfillTierReveal`, so a keeper can trigger a confidential check without ever learning the underlying encrypted value themselves.
2. **Direction-aware `FHE.select` cascades** for confidential decision logic (Confidential Derivatives uses this for SL/TP; CipherTrust uses it for the 3-tier confidential bond schedule and the tier-reveal encoding).
3. **Encrypted running-total pattern** (Confidential Derivatives accumulates lifetime PnL as a ciphertext across trades) -- CipherTrust's rolling trust score is the same idea applied to reliability instead of profitability.
4. **Confidential-token / composable-vault posture**: Pendex's GMX-style LP pool (stakers earn a share of trader losses) inspired CipherTrust's `InsurancePool`, where LPs earn a share of slashing penalties instead of trading losses.
5. **ERC-8004-style portable agent identity**, as pioneered by MARC Protocol for payment infrastructure -- CipherTrust's `AgentIdentityRegistry` applies the same idea of a shared, app-agnostic identity record, but keyed to *risk/underwriting* data instead of payment history.
6. **Soulbound public attestation**: no reviewed winner does this for a *risk tier*, but the general pattern of "reveal a coarse public signal, keep the precise ciphertext private" mirrors how Confidential Derivatives only reveals ITM/breach booleans, never full position data. CipherTrust's `ReputationBadge` applies that same coarse-signal principle to trust tiers.

## What makes CipherTrust distinct (not found in any reviewed project)

- Every reviewed FHE project scores or hides **human-directed financial positions or payments** (dark pools, derivatives, payroll/UBI, legal contracts, agent *payments*). None of them price **collateral/insurance requirements for the operational reliability of autonomous agents and robots** (completion rate, uptime, latency, error rate) -- a non-financial, telemetry-based confidential risk model.
- x402fhe's agent "reputation" is a byproduct of *payment history* (did the agent pay for what it used). CipherTrust's trust score is a multi-factor operational-performance score, generated by independent oracles/attestors, used specifically to size a collateral bond and trigger confidential slashing -- a materially different mechanism and use case (task/SLA underwriting, not payment credit).
- GitHub searches for `fhevm agent bonding insurance robot autonomous`, `fhevm ai model marketplace inference encrypted weights`, and `fhevm robotics drone swarm` continue to return zero matching repositories as of this update.
